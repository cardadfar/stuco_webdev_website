How do you debug when the code is not working?

When you load your file locally in your browser, you can right click on the page and select 'Inspect Elements' to get a console detailing any failed outputs that occurred (these consoles differ between browsers, but can be used in similar ways). You can also parse the html/css that the browser is trying to render to check if you website actually updated or if it's using a cached-version. We will talk more about these concepts in Lecture 5.

What does React do?

React is an easy way of building a user interface with minimal code. It abstracts away all the css when building common attributes like nav-bars and multi-tab panels. React is scalable and is used by many companies, but to understand React, you first have to understand css.

Still a little bit confused with the class inheritance and the ">".

Think of the > as a conditional. For example, if I have the class .cmu then every html element with that class will get the stylings associated with .cmu. Yet if I have .colleges > .cmu, then only html elements with BOTH the class name "cmu" that also have a direct parent with the class name "colleges" will get the stylings associated with it. We can have all elements wiith class "cmu" inheret their stylings from a general set of stylings, but elements specifically with the parent class "colleges" could get unique or additional stylings. This becomes very useful when you want a large group of elements to get one type of stylings, but then a subset of them should get additional styling.
